第 1 週
看懂現有流程 ＋ 討論分工
  • 看 AIS 入門影片（兩人一起）

第 2 週
設計每個 Agent 的規格

  • 評分 Agent 的四個維度邏輯草稿
  • 確認哪些會議要出考題、哪些不用（開關機制）
週報繳交 各 Agent 規格書（一頁就好，說清楚即可）
第 3 週
第一版 Agent 上線測試
  • 在 AIS 建立 Agent，用真實分享會內容測試
  • 完整跑一次：逐字稿 → 摘要 → 考題 → 信件草稿
  • 業務表單評分 Agent 第一版測試
週報繳交 第一版輸出截圖 ＋ 問題清單（哪裡不對）
第 4 週
🔍 中期 Check-in（Sophie 看成果）
  • 展示目前做好的東西
  • 討論後半段要改什麼、加什麼
  • 確認第二種會議類型要擴充哪個
中期匯報 Live Demo ＋ 問題說明 ＋ 後四週計畫
第 5 週
優化 ＋ 擴充第二種會議類型
  • 根據第四週回饋調整 Agent 輸出品質
  • 評分 Agent：測試多筆表單的評分一致性，完成燈號視覺化
  • 把架構套到第二種會議類型
週報繳交 評分一致性測試結果 ＋ 第二類型設計稿
第 6 週
整合測試 ＋ 寫使用說明
  • 模擬完整流程兩次（兩種會議類型各一次）
  • 記錄哪些地方還需要人工介入
  • 開始撰寫使用說明文件
週報繳交 完整測試紀錄 ＋ 使用說明初稿
第 7 週
做簡報 ＋ 彩排
  • 10 頁以內：問題是什麼 → 做了什麼 → 成果 → 以後能擴充到哪 → 省了多少時間
